JavaScript Objects vs. Ruby Hashes

How should we compare objects in JavasSript with hashes in Ruby?

November 13, 2015

If you've started to learn Ruby and then went on to learn JavaScript (or vice versa), you maybe struggling to compare and contrast the two languages. This post will compare and contrast Ruby hashes and JavaScript objects, and hopefully give you a better understanding of the two concepts.

First, let's define each term. A JavaScript object is a data container is allowed itself to have properties. These properties can be more like data (i.e. including a property called "name" with a value of "Natalie") or they can be functions (i.e. create a function called hello that prints "Hello!").

A hash in Ruby is also a data container. The information inside the hash can be made of strings, arrays, numbers and other hashes. Below is an example of a hash.

The information inside both the JavaScript object and the Ruby hash can be accessed in different ways. You can simply return a value you're looking for, add a new key/value pair, or delete a pair.

One of the main differences is that JavaScript object keys have to be variables (though they can point to any data type) and Ruby hash keys can be any data type (I mentioned some of these in the above definition of a hash).

So, while it is fair to see the similarities between JavaScript objects and Ruby hashes, they aren't technically the same thing.
