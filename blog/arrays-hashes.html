Arrays and Hashes

What they are and when to use them

October 25, 2015

Ruby uses arrays and hashes as ways to store other information. Arrays and hashes are called "containers," which helps make sense of them to those without technical knowledge; simply put, an array or hash contains other information.

Arrays are ordered containers of objects. Objects can be anything - text, numbers, another array or hash. They can look like the below.

My_array1 = ["this", "is", "my", "array"]

My_array2 = [100, 5, 101, 4444]

My_array3 = [ [5, -3, 0], "this", 33 ]

Hashes, on the other hand, generally look more like this:

my_hash = {

:name => "Natalie",

:date => "Today",

:age => 30,

:array => [1, 55, -4]

}

As you can see, a hash stores two pieces of information that relate to each other: keys and values. :name is a key and its value is "Natalie". In the array, the key is the implicit. For example, in My_array1 above, the key for "this" is 0 (the number associated with the first position in an array), the key for "is" is 1, the key for "my" is 2, and the key for "array" is 3.

So, if you want to associate a value with a specific key (i.e. associate a specific password with a username or a date with an event), you will want to use a hash. If that information doesn't matter, you can use an array. And if you want to easily retrieve the data based on order, an array ususally makes more sense - you can ask it for the first nth terms and know what you're going to get.

Hashes are better used when you don't want to have only a number as the indication for a value. Hashes are better when you want to associate information with other information that you already know.
