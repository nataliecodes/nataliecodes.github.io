A Comparison of Looping in Ruby and JavaScript

This cheat sheet is meant to compare and contrast looping in Ruby and JavaScript. First, remember the definition of looping or iteration. Looping means that a certain instruction is done over and over until a different condition is met.

Both languages have many ways to accomplish a loop. In the chart below, you will see that Ruby looks "cleaner" because JavaScript has more text and lines of code. This is because Ruby is more implicit, meaning it can better understand your intention. JavaScript makes you spell everything out more clearly.

JavaScript

for (var i = 1; i < 11; i++) {
console.log(i*2);
}

The for loop has three parameters:

  • var i = 1 = The state of the variable. Variable i is set equal to one.
  • i < 11 = The conditions under which we want to loop. Loop through i until it's less than 11. Since we declare i as 1, the final iteration of i will be an integer -- 10.
  • i++ = What to do after the loop is iterated once.

var i = 0;
while (i <= 5) {
console.log("A line of code.");
i += 1
}

  • JavaScript also has "while" loops. This example sets a variable i outside of the method.
  • The condition is on the next line; while i is less than 5, print the code.
  • The counter is then below, and the loop ends after the line is printed 6 times.

Ruby

10.times do |n|
puts i*2
end

Less has to be declared in Ruby.

  • We don't have to set a variable here. We just say "do this 10 times."
  • There is no end condition necessary here. But, if you write the loop another way, you could have a more direct translation from JavaScript.
  • There is no iterator increase or change here, because we've declared it already.

i = 0
while i <= 5
p "A line of code."
i += 1
end

  • This Ruby code looks very similar to the JavaScript on the left.
  • This is a way to show that they two languages are similar.
  • The only real differences are: Ruby has less punctuation and printing is a different command.

In the simplest terms, JavaScript requires more upfront information. Also, Ruby is more flexible because you can loop in may different ways. While there are a couple of different ways to use the for loop in JavaScript, Ruby offers more functionality.
